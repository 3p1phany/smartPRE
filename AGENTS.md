## Project Overview

smartPRE is a CPU/memory co-simulation research project combining two submodules:
- **champsim-la** — A modified ChampSim cycle-level CPU simulator with LoongArch ISA support
- **dramsim3** — DRAMSim3 detailed DRAM memory system simulator


## Build Commands

DRAMSim3 must be built before ChampSim (ChampSim links against `libdramsim3.so`).

### Build DRAMSim3 (CMake)

```bash
cd dramsim3
mkdir -p build && cd build && cmake ..
make -j8
```

This produces `libdramsim3.so` in the build directory. An alternative plain Makefile exists at `dramsim3/Makefile` for environments without CMake 3.0+, producing `libdramsim3.so` directly in the dramsim3 root.

### Build ChampSim-LA

```bash
cd champsim-la
python3 config.sh champsim_config.json   # generates Makefile rules, headers, and instantiation code
make -j8
```

Output: `bin/champsim`

`config.sh` is a Python 3 script (despite the `.sh` extension) that reads a JSON config and generates:
- `inc/champsim_constants.h` — compile-time constants
- `src/core_inst.cc` — component instantiation
- `inc/cache_modules.inc`, `inc/ooo_cpu_modules.inc` — module includes
- The `Makefile` itself (overwrites it with correct module targets)


### Runtime requirement

DRAMSim3 is dynamically linked. Before running simulations:
```bash
export LD_LIBRARY_PATH=/root/data/smartPRE/dramsim3:$LD_LIBRARY_PATH
```

### Clean

```bash
cd champsim-la && make clean
cd dramsim3 && make clean        # plain Makefile
cd dramsim3/build && make clean  # CMake build
```


## Architecture

### ChampSim-LA (`champsim-la/`)

The CPU simulator is a cycle-accurate out-of-order core model driven by instruction traces.

| Directory | Role |
|-----------|------|
| `src/` | Core simulation: `main.cc` (entry, simulation loop), `ooo_cpu.cc` (pipeline stages), `cache.cc` (cache hierarchy), `dram_controller.cc` (memory interface), `ptw.cc` (page table walker), `tracereader.cc` (trace input) |
| `inc/` | Headers. `dramsim3_wrapper.hpp` bridges ChampSim to DRAMSim3. Many headers are auto-generated by `config.sh`. |
| `loongarch/` | LoongArch ISA decoder (`decode-new.c`, `ir2.c`, `ins.c`) |
| `branch/` | Branch predictors (e.g., `tage-sc-l/`) |
| `btb/` | Branch target buffers (e.g., `basic_btb/`) |
| `prefetcher/` | Prefetcher modules (`no`, `stride`, `berti`, `spp_dev`, `la864`, etc.) |
| `replacement/` | Cache replacement policies (`lru/`) |
| `dramsim3_configs/` | DRAM timing configs (e.g., `DDR5_64GB_4ch_4800.ini`) |

**Module system**: Prefetchers, replacement policies, branch predictors, and BTBs are compiled as static archives (`obj/*.a`) and linked into the binary. Each module's functions are renamed via `-D` flags to avoid symbol collisions, allowing multiple modules to coexist.

### DRAMSim3 (`dramsim3/`)

| Directory | Role |
|-----------|------|
| `src/` | Simulator core: `dram_system.cc`, `controller.cc`, `command_queue.cc`, `channel_state.cc`, `bankstate.cc`, `configuration.cc`, `memory_system.cc` |
| `configs/` | 30+ DRAM timing configurations (DDR3, DDR4, DDR5 variants) |
| `ext/` | Vendored dependencies: `fmt`, `inih`, `json`, `SuperLU_MT` (optional thermal) |
| `tests/` | Unit tests (`test_config.cc`, `test_dramsys.cc`, `test_hmcsys.cc`) |

### Configuration

Simulation parameters are defined in JSON config files (e.g., `champsim_config.json`):
- Core params: ROB size, widths, frequencies, branch predictor/BTB selection
- Cache hierarchy: L1I, L1D, L2C, LLC, TLBs — sizes, associativity, latency, prefetcher/replacement selection
- Physical memory: DRAMSim3 config path, channel count, timing
- Virtual memory: address space size, page table levels

Multi-core configs: `champsim_config_4c.json` (4-core), `champsim_config_8C.json` (8-core).

### Simulation workflow

Simulations are trace-driven. Traces are ChampSim-format instruction traces of SPEC2017, LIGRA, PARSEC, and CloudSuite workloads. Benchmark metadata is in `benchmarks.tsv` and `benchmarks_selected.tsv`.

Key scripts in `champsim-la/`:
- `scripts/run_benchmarks.sh` — **primary run script** for running benchmarks listed in `benchmarks_selected.tsv`. Always use this as the basis for custom experiment scripts.
- `batch_run/run.py` — parallel task management for batch runs
- `scripts/summarize_ipc.py`, `scripts/compare_ipc.py` — parse and compare IPC results
- `scripts/run_selected_slices.sh` — quick functional correctness testing (smoke tests) only, not for performance evaluation

Results are stored in `results/` organized by configuration name (e.g., `results/GS_1c/`).

## Key Technical Details

- ChampSim uses C++17; DRAMSim3 uses C++11
- ChampSim Makefile is auto-generated — do not edit it directly; modify `config.sh` or the JSON config instead
- The `config.sh` script must be re-run after changing the JSON config or adding/removing modules
- DRAM controller integration is through `inc/dramsim3_wrapper.hpp`, which wraps DRAMSim3's `MemorySystem` API for cycle-accurate co-simulation
- The LDLIBS path in the generated Makefile is hardcoded to the DRAMSim3 location — if the repo moves, re-run `config.sh`
